- [ ] 1교시 실습 완료 (n8n Cloud 계정 활성화)

## 사전 준비: 수강생 명단 Google Sheets 만들기

### Step 1: Google Sheets 샘플 데이터 생성

1. 브라우저 새 탭에서 `sheets.google.com` 접속

2. **"빈 스프레드시트"** 클릭하여 새 시트 생성

3. 파일 이름을 **"수강생_명단"** 으로 변경 (좌측 상단 클릭)

4. 아래 데이터를 입력합니다 (1행이 헤더):

   | A열(이름) | B열(이메일) | C열(과정) | D열(납부여부) | E열(연락처) |
   |-----------|-------------|-----------|--------------|-------------|
   | 이름 | 이메일 | 과정 | 납부여부 | 연락처 |
   | 홍길동 | hong@test.com | AI입문 | 완료 | 010-1234-5678 |
   | 김영희 | kim@test.com | AI입문 | 미납 | 010-2345-6789 |
   | 박철수 | park@test.com | 데이터분석 | 완료 | 010-3456-7890 |
   | 이지은 | lee@test.com | AI입문 | 미납 | 010-4567-8901 |
   | 최민수 | choi@test.com | 데이터분석 | 완료 | 010-5678-9012 |

   > ⚠️ **중요**: 1행(헤더)의 열 이름은 정확히 위와 같이 입력하세요. n8n에서 이 이름으로 데이터를 참조합니다.

5. 데이터 입력 후 시트가 자동 저장되는지 확인합니다

---

## 실습 2: Google Sheets 연동

### 목표
n8n에서 Google Sheets의 데이터를 읽어오고 새 데이터를 추가합니다.

---

### Step 1: Google OAuth 인증 (Credential) 등록

1. n8n 좌측 메뉴에서 **"Credentials"** 클릭

2. 우측 상단 **"+ Add Credential"** 클릭

3. 검색창에 **"Google Sheets"** 입력

4. **"Google Sheets OAuth2 API"** 선택

5. **"Sign in with Google"** 버튼 클릭

6. Google 계정 선택 화면이 나타납니다
   - 본인 Google 계정을 선택합니다
   - "이 앱은 확인되지 않았습니다" 메시지가 나오면 **"고급" → "n8n(으)로 이동 (안전하지 않음)"** 클릭

7. 권한 요청 화면에서 **"모두 선택"** 또는 필요한 항목 체크:
   - Google Sheets 보기, 수정, 만들기, 삭제
   - Google Drive 파일 보기 및 관리

8. **"계속"** 버튼 클릭

9. 인증 성공 후 n8n으로 돌아옵니다

10. Credential 이름을 **"My Google"** 으로 지정

11. **"Save"** 버튼 클릭

> ✅ **확인**: "Connection tested successfully" 메시지가 보이면 성공!

---

### Step 2: Google Sheets 읽기 워크플로우

1. 좌측 메뉴 **"Workflows"** → **"+ Add Workflow"** 클릭

2. 워크플로우 이름: **"실습2_Sheets읽기쓰기"**

3. **Manual Trigger** 노드 추가 (Tab 키 → "Manual" 검색)

4. Manual Trigger 우측 **[+]** 클릭 → **"Google Sheets"** 검색 → 선택

5. Google Sheets 노드 설정:
   - **Resource**: Sheet
   - **Operation**: **"Read Rows"** 선택
   - **Credential**: "My Google" 선택
   - **Document**: 드롭다운에서 **"수강생_명단"** 선택
     > 목록에 안 보이면 검색창에 "수강생" 입력
   - **Sheet**: **"Sheet1"** 선택 (기본 시트)

6. **"Back to canvas"** 클릭하여 캔버스로 돌아갑니다

7. 상단 **"Test workflow"** (▶) 클릭

8. Google Sheets 노드를 클릭 → **"Output"** 탭 확인

9. 아래와 같은 형태의 데이터가 보여야 합니다:

   ```json
   [
     {
       "이름": "홍길동",
       "이메일": "hong@test.com",
       "과정": "AI입문",
       "납부여부": "완료",
       "연락처": "010-1234-5678"
     },
     {
       "이름": "김영희",
       "이메일": "kim@test.com",
       ...
     }
   ]
   ```

> ✅ **확인**: 5명의 수강생 데이터가 모두 보이면 성공!

---

### Step 3: Google Sheets 쓰기 (새 행 추가)

1. 같은 워크플로우에서 Google Sheets(Read) 노드 뒤에 새 노드 추가

2. **"Google Sheets"** 노드를 하나 더 추가합니다

3. 설정:
   - **Operation**: **"Append Row"** 선택
   - **Credential**: "My Google"
   - **Document**: "수강생_명단"
   - **Sheet**: "Sheet1"

4. **Mapping Mode**: "Map Automatically" 선택 시, 아래 **"Columns to Send"** 에 값 지정:
   - **이름**: `테스트학생`
   - **이메일**: `test@example.com`
   - **과정**: `AI입문`
   - **납부여부**: `완료`
   - **연락처**: `010-0000-0000`

5. **"Test workflow"** 실행

6. Google Sheets 브라우저 탭으로 이동하여 새 행이 추가되었는지 확인합니다

> ✅ **확인**: "테스트학생" 데이터가 시트 마지막 행에 추가되면 성공!
> 
> 🧹 **정리**: 테스트 데이터 행은 확인 후 삭제해도 됩니다.

## 실습 3: 데이터 변환 & 조건 분기

### 목표
Edit Fields로 데이터를 가공하고, IF 노드로 조건에 따른 분기를 구현합니다.

---

### Step 1: Edit Fields 노드로 데이터 가공

1. 새 워크플로우 생성: **"실습3_데이터변환"**

2. **Manual Trigger** → **Google Sheets (Read Rows)** 연결
   - 앞의 실습2와 동일한 설정

3. Google Sheets 뒤에 **"Edit Fields (Set)"** 노드 추가

4. Edit Fields 노드 설정 — **"Add Field"** 를 클릭하여 아래 필드들을 추가합니다:

   | 필드 이름 | 타입 | 값 (Value) | 설명 |
   |-----------|------|------------|------|
   | `수강생이름` | String | `{{ $json.이름 }}` | 필드 이름 변경 |
   | `수강생이메일` | String | `{{ $json.이메일 }}` | 필드 이름 변경 |
   | `과정명` | String | `{{ $json.과정 }}` | 필드 이름 변경 |
   | `납부상태` | String | `{{ $json.납부여부 }}` | 필드 이름 변경 |
   | `등록일` | String | `{{ $now.format('YYYY-MM-DD') }}` | 현재 날짜 자동 추가 |

5. **"Include Other Input Fields"** 를 **Off** 로 설정
   > 원본 필드를 제외하고 새로 정의한 필드만 출력합니다

6. **"Test workflow"** 실행 → Edit Fields 노드의 Output 확인

   ```json
   {
     "수강생이름": "홍길동",
     "수강생이메일": "hong@test.com",
     "과정명": "AI입문",
     "납부상태": "완료",
     "등록일": "2026-03-22"
   }
   ```

> ✅ **확인**: 5개 필드가 새 이름으로 변환되고 등록일이 추가되면 성공!

---

### Step 2: IF 노드로 조건 분기

1. Edit Fields 노드 뒤에 **"IF"** 노드 추가
   - Tab 키 → "IF" 검색 → 선택

2. IF 노드 설정:
   - **Conditions**: "Add Condition" 클릭
   - **Condition Type**: **String**
   - **Value 1** (좌측): `{{ $json.납부상태 }}`
     > 입력란을 클릭하면 "Fixed" / "Expression" 토글이 있습니다
     > **"Expression"** 으로 전환한 후 위 값을 입력합니다
   - **Operation**: **"is equal to"**
   - **Value 2** (우측): `완료`
     > 이 값은 "Fixed" 모드로 직접 입력합니다

3. 설정이 올바르면 IF 노드에 두 개의 출력이 생깁니다:

   ```
                    ┌──── TRUE  (납부완료)
   Edit Fields ──▶ IF
                    └──── FALSE (미납)
   ```

4. **"Test workflow"** 실행

5. IF 노드를 클릭하여 결과 확인:
   - **TRUE 출력** (Output 0): 납부 "완료"인 수강생 3명
   - **FALSE 출력** (Output 1): 납부 "미납"인 수강생 2명

> ✅ **확인**: TRUE에 3명, FALSE에 2명이 분류되면 성공!

---

### Step 3: Switch 노드 (다중 조건 분기)

1. IF 노드 대신 **"Switch"** 노드를 사용해봅니다

2. Switch 노드 설정:
   - **Mode**: "Rules"
   - **Rules**: 
     - Rule 0: `{{ $json.과정명 }}` **is equal to** `AI입문` → Output 0
     - Rule 1: `{{ $json.과정명 }}` **is equal to** `데이터분석` → Output 1
   - **Fallback Output**: 나머지

3. 결과: 과정별로 수강생이 분류됩니다

   ```
                       ┌──── Output 0: AI입문 수강생
   Edit Fields ──▶ Switch
                       └──── Output 1: 데이터분석 수강생
   ```

> 💡 **TIP**: IF는 2갈래, Switch는 3갈래 이상 분기할 때 사용합니다.

## 실습 4: 수강생 상태별 이메일 자동 발송

### 목표
납부 완료 수강생에게는 환영 이메일, 미납 수강생에게는 납부 안내 이메일을 자동 발송합니다.

---

### Step 1: 워크플로우 전체 구조

```
Manual Trigger → Google Sheets (Read) → IF (납부여부)
                                          ├── TRUE → Gmail (환영 이메일)
                                          └── FALSE → Gmail (납부 안내)
```

1. 새 워크플로우 생성: **"실습4_상태별이메일발송"**

2. **Manual Trigger** 추가

3. **Google Sheets** (Read Rows) 추가 → "수강생_명단" 시트 연동

4. **IF** 노드 추가 → 조건: `{{ $json.납부여부 }}` is equal to `완료`

---

### Step 2: Gmail 인증 설정

1. n8n 좌측 **"Credentials"** → **"+ Add Credential"**

2. **"Gmail OAuth2 API"** 검색 → 선택

3. **"Sign in with Google"** → Google 계정 선택 → 권한 허용

4. Credential 이름: **"My Gmail"** → Save

---

### Step 3: 환영 이메일 (납부 완료)

1. IF 노드의 **TRUE** 출력에 **"Gmail"** 노드 연결

2. Gmail 노드 설정:
   - **Credential**: "My Gmail"
   - **Operation**: "Send"
   - **To**: Expression 모드로 전환 후 입력
     ```
     {{ $json.이메일 }}
     ```
   - **Subject**:
     ```
     [{{ $json.과정 }}] 수강 신청이 완료되었습니다 🎉
     ```
   - **Email Type**: HTML
   - **Message** (HTML 본문):
     ```html
     <h2>안녕하세요, {{ $json.이름 }}님!</h2>
     <p><strong>{{ $json.과정 }}</strong> 과정에 수강 신청이 완료되었습니다.</p>
     <h3>📌 수업 안내</h3>
     <ul>
       <li>과정명: {{ $json.과정 }}</li>
       <li>수강료 납부: ✅ 완료</li>
       <li>문의: education@example.com</li>
     </ul>
     <p>수업에서 뵙겠습니다. 감사합니다! 🙏</p>
     ```

---

### Step 4: 납부 안내 이메일 (미납)

1. IF 노드의 **FALSE** 출력에 **"Gmail"** 노드 추가 (두 번째 Gmail)

2. Gmail 노드 설정:
   - **To**: `{{ $json.이메일 }}`
   - **Subject**:
     ```
     [{{ $json.과정 }}] 수강료 납부를 완료해주세요
     ```
   - **Message** (HTML 본문):
     ```html
     <h2>안녕하세요, {{ $json.이름 }}님</h2>
     <p><strong>{{ $json.과정 }}</strong> 과정에 신청해주셔서 감사합니다.</p>
     <p>수강료 납부가 아직 완료되지 않았습니다.</p>
     <h3>💳 납부 안내</h3>
     <ul>
       <li>입금 계좌: 국민은행 123-456-789012</li>
       <li>입금자명: {{ $json.이름 }}</li>
       <li>납부 기한: 수업 시작 3일 전까지</li>
     </ul>
     <p>납부 완료 후 확인 이메일이 발송됩니다. 감사합니다!</p>
     ```

---

### Step 5: 테스트 실행

1. **"Test workflow"** (▶) 클릭

2. 실행 완료 후 각 노드의 Output을 확인합니다:
   - IF TRUE: 홍길동, 박철수, 최민수 (3명)
   - IF FALSE: 김영희, 이지은 (2명)

3. 본인 Gmail 받은편지함에서 이메일 수신을 확인합니다

> ⚠️ **주의**: 테스트 시 수강생 이메일이 실제 주소가 아니므로 이메일 발송이 실패할 수 있습니다.
> 테스트용으로 본인 이메일 주소를 시트에 1개 넣어두면 실제 수신을 확인할 수 있습니다.

> ✅ **확인**: 워크플로우가 에러 없이 실행되고 Gmail 노드 Output에 발송 결과가 보이면 성공!

## 에러 핸들링 & 디버깅 팁

### 자주 발생하는 오류

| 오류 | 원인 | 해결 방법 |
|------|------|-----------|
| 401 Unauthorized | 인증 만료 | Credential 삭제 후 재등록 |
| 403 Forbidden | 권한 부족 | Google 권한 설정에서 Sheets/Gmail 허용 확인 |
| "No data found" | 빈 시트 | 시트에 데이터가 있는지 확인, 헤더(1행) 존재 확인 |
| 필드명 불일치 | 대소문자 | 시트 헤더와 Expression의 필드명이 정확히 동일한지 확인 |
| Expression 에러 | 구문 오류 | `{{ }}` 안의 표현식 오타 확인, $json. 뒤 필드명 확인 |

### 디버깅 순서

1. **실행 로그 확인**: 좌측 "Executions" 메뉴에서 실행 이력 확인
2. **실패 노드 확인**: 빨간색 표시된 노드를 더블클릭하여 에러 메시지 읽기
3. **Input/Output 확인**: 각 노드의 Input(입력)과 Output(출력) 데이터 비교
4. **Pin Data 활용**: 정상 동작하는 노드의 Output을 Pin하면 이후 테스트 시 재사용 가능
