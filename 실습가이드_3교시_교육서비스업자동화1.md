## 실습 5: 수강 신청 자동 처리 시스템

### 목표
수강생이 웹폼을 제출하면 → Google Sheets 저장 → 중복 체크 → 확인 이메일 → 관리자 알림까지 자동 처리

### 전체 워크플로우 구조
```
Webhook Trigger → Edit Fields → Google Sheets (Append)
                                       ↓
                              Google Sheets (Search)
                                       ↓
                                    IF (중복?)
                              ┌────────┴────────┐
                     FALSE (신규)            TRUE (중복)
                         ↓                       ↓
                  Gmail (환영메일)         Webhook Response
                         ↓                  ("이미 신청됨")
                  Slack (관리자알림)
                         ↓
                  Webhook Response
                    ("신청 완료")
```

---

### Step 1: 사전 준비 — 신청자 DB 시트 만들기

1. Google Sheets에서 새 스프레드시트 생성: **"수강신청_DB"**

2. 1행(헤더)에 아래 열을 입력합니다:

   | A | B | C | D | E |
   |---|---|---|---|---|
   | 이름 | 이메일 | 과정명 | 신청일시 | 연락처 |

3. 데이터는 비워둡니다 (Webhook으로 자동 추가될 예정)

---

### Step 2: Webhook Trigger 설정

1. n8n에서 새 워크플로우 생성: **"실습5_수강신청자동처리"**

2. 노드 추가 → **"Webhook"** 검색 → **"Webhook"** 선택

3. Webhook 노드 설정:
   - **HTTP Method**: `POST`
   - **Path**: `enrollment`
   - **Authentication**: `None`
   - **Response Mode**: `Last Node` (마지막 노드의 결과를 응답으로 반환)

4. **"Test URL"** 을 확인합니다 (형식 예시):
   ```
   https://your-instance.app.n8n.cloud/webhook-test/enrollment
   ```
   > 이 URL을 메모장에 복사해둡니다

5. **"Listen for test event"** 버튼을 클릭합니다 (테스트 대기 모드)

---

### Step 3: Webhook 테스트 데이터 보내기

**방법 A: 브라우저 주소창 사용 (간단 테스트)**
- 이 방법은 GET 요청만 가능하므로 POST 테스트에는 적합하지 않습니다

**방법 B: n8n 내 다른 워크플로우에서 테스트**

1. n8n에서 **새 탭**을 열어 새 워크플로우를 하나 더 만듭니다: **"Webhook테스트용"**

2. Manual Trigger → **HTTP Request** 노드 추가

3. HTTP Request 노드 설정:
   - **Method**: `POST`
   - **URL**: 위에서 복사한 Test URL
   - **Body Content Type**: `JSON`
   - **Body Parameters** — "Add Parameter" 클릭하여 아래 입력:

     | Name | Value |
     |------|-------|
     | name | 테스트학생 |
     | email | test@example.com |
     | course | AI입문 |
     | phone | 010-9999-8888 |

4. **원래 워크플로우 탭**으로 돌아가서 Webhook이 "Listening..." 상태인지 확인

5. **테스트용 워크플로우 탭**에서 "Test workflow" 실행

6. 원래 워크플로우로 돌아가면 Webhook 노드에 데이터가 수신된 것을 확인:

   ```json
   {
     "body": {
       "name": "테스트학생",
       "email": "test@example.com",
       "course": "AI입문",
       "phone": "010-9999-8888"
     },
     "headers": { ... },
     "query": { ... }
   }
   ```

> ✅ **확인**: Webhook 노드 Output에 body 데이터가 보이면 성공!

---

### Step 4: 데이터 정리 (Edit Fields)

1. Webhook 노드 뒤에 **"Edit Fields (Set)"** 노드 추가

2. 설정 — 필요한 필드만 추출:

   | Name | Value (Expression) |
   |------|-------------------|
   | `이름` | `{{ $json.body.name }}` |
   | `이메일` | `{{ $json.body.email }}` |
   | `과정명` | `{{ $json.body.course }}` |
   | `연락처` | `{{ $json.body.phone }}` |
   | `신청일시` | `{{ $now.format('YYYY-MM-DD HH:mm:ss') }}` |

3. **"Include Other Input Fields"**: Off

---

### Step 5: Google Sheets에 저장

1. Edit Fields 뒤에 **"Google Sheets"** 노드 추가

2. 설정:
   - **Operation**: `Append Row`
   - **Document**: "수강신청_DB"
   - **Sheet**: "Sheet1"
   - **Mapping Mode**: "Map Automatically"
     > 필드 이름이 시트 헤더와 동일하면 자동 매핑됩니다

---

### Step 6: 중복 신청 체크

1. Google Sheets (Append) 뒤에 **"Google Sheets"** 노드 하나 더 추가

2. 설정:
   - **Operation**: `Search Rows` (또는 Read Rows + IF)
   - **Document**: "수강신청_DB"
   - **Sheet**: "Sheet1"
   - **Filter**: 
     - Column: `이메일`
     - Value: `{{ $json.이메일 }}`

3. Google Sheets (Search) 뒤에 **"IF"** 노드 추가

4. IF 조건:
   - Value 1: `{{ $json.length }}` (또는 `{{ $input.all().length }}`)
   - Operation: `is greater than`
   - Value 2: `1`
   > 검색 결과가 2건 이상이면 중복 (방금 저장한 1건 + 기존 1건)

---

### Step 7: 확인 이메일 & 관리자 알림

**신규 신청 (IF FALSE):**

1. IF의 **FALSE** 출력에 **Gmail** 노드 연결

2. Gmail 설정:
   - **To**: `{{ $json.이메일 }}`
   - **Subject**: `[{{ $json.과정명 }}] 수강 신청이 완료되었습니다`
   - **Message**:
     ```html
     <h2>{{ $json.이름 }}님, 환영합니다!</h2>
     <p><strong>{{ $json.과정명 }}</strong> 수강 신청이 정상적으로 접수되었습니다.</p>
     <p>신청일시: {{ $json.신청일시 }}</p>
     <p>수업 관련 상세 안내는 별도 이메일로 발송드리겠습니다.</p>
     ```

3. Gmail 뒤에 **Slack** 노드 추가 (선택사항 — Slack이 없으면 Gmail로 대체):
   - **Channel**: `#enrollment-alerts` (또는 테스트 채널)
   - **Text**: 
     ```
     🆕 새 수강 신청!
     • 이름: {{ $json.이름 }}
     • 과정: {{ $json.과정명 }}
     • 이메일: {{ $json.이메일 }}
     • 시간: {{ $json.신청일시 }}
     ```

4. 마지막에 **"Respond to Webhook"** 노드 추가:
   - **Response Body**: JSON
     ```json
     { "status": "success", "message": "수강 신청이 완료되었습니다." }
     ```

**중복 신청 (IF TRUE):**

5. IF의 **TRUE** 출력에 **"Respond to Webhook"** 노드 연결:
   - **Response Code**: 409
   - **Response Body**:
     ```json
     { "status": "duplicate", "message": "이미 동일한 이메일로 신청된 기록이 있습니다." }
     ```

---

### Step 8: 전체 테스트

1. Webhook 노드를 클릭 → **"Listen for test event"** 클릭

2. 테스트용 워크플로우에서 HTTP Request 실행

3. 전체 흐름이 정상 동작하는지 확인:
   - Google Sheets에 데이터 추가 여부
   - 이메일 발송 여부
   - 중복 시 409 응답 여부 (같은 이메일로 재요청)

> ✅ **산출물**: Webhook 기반 수강 신청 자동 처리 워크플로우 완성!

## 실습 6: 수업 리마인더 자동 발송 시스템

### 목표
매일 오전 9시에 자동 실행되어 내일 수업이 있는 수강생에게 리마인더를 발송합니다.

---

### Step 1: 수업 일정 시트 만들기

1. Google Sheets에서 새 시트 생성: **"수업일정"**

2. 데이터 입력:

   | 과정명 | 수업날짜 | 수업시간 | 강의실 | 강사 |
   |--------|----------|----------|--------|------|
   | AI입문 | 2026-03-23 | 09:00 | 301호 | 김강사 |
   | 데이터분석 | 2026-03-24 | 14:00 | 302호 | 이강사 |
   | AI입문 | 2026-03-25 | 09:00 | 301호 | 김강사 |

   > ⚠️ 수업날짜는 **내일 날짜**로 최소 1개 넣어주세요 (테스트용)

---

### Step 2: 워크플로우 구성

1. 새 워크플로우: **"실습6_수업리마인더"**

2. **Schedule Trigger** 노드 추가:
   - **Trigger Interval**: "Days"
   - **Days Between Triggers**: 1
   - **Trigger at Hour**: 9
   - **Trigger at Minute**: 0
   > 매일 오전 9시 실행 (테스트 시에는 Manual Trigger를 병렬로 추가하면 편합니다)

3. **Google Sheets** (Read Rows) → "수업일정" 시트 전체 읽기

4. **IF** 노드 추가:
   - Value 1: `{{ $json.수업날짜 }}`
   - Operation: `is equal to`
   - Value 2: `{{ $now.plus(1, 'day').format('YYYY-MM-DD') }}`
   > 내일 날짜와 일치하는 수업만 필터링

5. IF **TRUE** 출력 → **Google Sheets** (Read Rows) → "수강생_명단" 시트
   - 해당 과정의 수강생 목록 조회
   - Filter 또는 후속 IF 노드로 과정명 매칭

6. → **Gmail** 노드:
   - **To**: `{{ $json.이메일 }}`
   - **Subject**: `[리마인더] 내일 {{ $json.과정명 }} 수업이 있습니다`
   - **Message**:
     ```html
     <h2>{{ $json.이름 }}님, 내일 수업이 있습니다!</h2>
     <ul>
       <li>📚 과정: {{ $json.과정명 }}</li>
       <li>🕐 시간: {{ $json.수업시간 }}</li>
       <li>📍 장소: {{ $json.강의실 }}</li>
       <li>👨‍🏫 강사: {{ $json.강사 }}</li>
     </ul>
     <p>준비물을 잊지 마시고, 10분 전까지 입실해주세요!</p>
     ```

---

### Step 3: 테스트

1. Manual Trigger를 Schedule Trigger와 병렬로 연결해둡니다
2. Manual Trigger로 테스트 실행
3. 내일 날짜에 해당하는 수업의 수강생에게 이메일이 발송되는지 확인

> ✅ **산출물**: 스케줄 기반 수업 리마인더 발송 워크플로우
