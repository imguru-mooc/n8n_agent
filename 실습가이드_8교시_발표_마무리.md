## 팀별 프로젝트 발표 (30분)

### 발표 형식

| 항목 | 시간 | 내용 |
|------|------|------|
| 발표 | 5분 | 문제 → 해결 → 데모 → 기대효과 |
| Q&A | 2분 | 동료 질의응답 |
| 전환 | 1분 | 다음 팀 준비 |

### 발표 구성 가이드

**1. 문제 정의 (1분)**
- 어떤 업무를 자동화했는지
- 기존에 얼마나 시간이 걸렸는지
- 왜 자동화가 필요했는지

**2. 해결 방안 (1.5분)**
- 워크플로우 전체 구조 설명 (스크린샷)
- 핵심 노드 소개 (어떤 노드를 왜 사용했는지)
- AI 연동 부분 하이라이트

**3. 라이브 데모 (1.5분)**
- 실제 워크플로우 실행 시연
- 데이터 입력 → 처리 과정 → 결과 확인
- Google Sheets 데이터 변화, 이메일 수신 등

**4. 기대 효과 (1분)**
- 시간 절감 효과 (예: 주 3시간 → 자동)
- 오류 감소 효과
- 확장 가능성 (추후 추가할 기능)

## 실무 적용 가이드

### 워크플로우 운영 모범 사례

#### 활성화(Active) 전 체크리스트

워크플로우를 실제 운영에 투입하기 전에 반드시 확인하세요:

- [ ] 모든 Credential이 유효한지 확인 (Google, OpenAI 등)
- [ ] 테스트 데이터로 최소 3회 이상 성공적으로 실행됨
- [ ] Webhook URL을 Production URL로 변경 (Test URL → Production)
- [ ] Error Workflow 설정 완료 (에러 시 관리자 알림)
- [ ] 워크플로우 이름과 설명(Notes)을 명확하게 작성
- [ ] 불필요한 테스트 노드 제거 또는 비활성화
- [ ] API 사용량 모니터링 설정 (OpenAI 비용 관리)

#### Error Workflow 설정 방법

1. 새 워크플로우: **"에러알림"** 생성
2. **Error Trigger** 노드 추가
3. → **Gmail** 또는 **Slack** 노드:
   - 에러 메시지, 워크플로우 이름, 발생 시간 전달
4. 각 운영 워크플로우 설정에서:
   - Settings → **"Error Workflow"** → "에러알림" 워크플로우 지정

#### 워크플로우 네이밍 규칙 (추천)

```
[팀/부서]_[업무명]_v[버전]

예시:
교육팀_수강신청처리_v1
교육팀_수업리마인더_v2
교육팀_피드백수집_v1
```

### 고급 기능 소개

#### Sub-workflow (워크플로우 내 워크플로우)
- 자주 반복되는 로직을 별도 워크플로우로 분리
- **Execute Workflow** 노드로 호출
- 예: "이메일 발송" 로직을 Sub-workflow로 만들어 여러 곳에서 재사용

#### n8n AI Agent 노드
- 단순 API 호출을 넘어 AI가 도구를 자율적으로 선택·사용
- 예: "수강 현황을 조회해서 요약해줘" → AI가 알아서 Sheets 조회 → 요약 생성

#### 자체 호스팅 (Self-hosting)
- Docker로 사내 서버에 n8n 설치 가능
- 데이터가 외부로 나가지 않음 (보안 우수)
- 워크플로우/실행 횟수 무제한
