## 실습 1: Hello World 워크플로우

### 목표
n8n Cloud에 가입하고, 가장 기본적인 워크플로우를 만들어 실행해봅니다.

---

### Step 1: n8n Cloud 가입

1. Chrome 브라우저를 열고 주소창에 입력합니다
   ```
   https://app.n8n.cloud
   ```

2. **"Sign up free"** 버튼을 클릭합니다

3. 가입 정보를 입력합니다
   - **First Name**: 본인 이름
   - **Last Name**: 본인 성
   - **Email**: 본인 이메일 주소
   - **Password**: 8자 이상 비밀번호

4. **"Create account"** 버튼을 클릭합니다

5. 이메일 인증
   - 입력한 이메일 받은편지함을 확인합니다
   - n8n에서 보낸 인증 메일의 **"Verify email"** 링크를 클릭합니다

6. 인증 완료 후 n8n 대시보드에 자동으로 이동됩니다

> ✅ **확인**: 화면 좌측에 "Workflows", "Credentials", "Executions" 메뉴가 보이면 성공!

---

### Step 2: n8n 워크플로우 편집기 UI 익히기

1. 좌측 메뉴에서 **"Workflows"** 클릭

2. 우측 상단 **"+ Add Workflow"** 버튼 클릭

3. 새 워크플로우 편집기가 열립니다. 화면 구성을 확인하세요:

   ```
   ┌─────────────────────────────────────────────┐
   │  워크플로우 이름 (클릭하여 수정)    [Save] [▶] │
   ├─────────────────────────────────────────────┤
   │                                             │
   │         캔버스 (Canvas)                       │
   │         노드를 배치하는 작업 영역                │
   │                                             │
   │              [ + ]  ← 노드 추가 버튼           │
   │                                             │
   ├─────────────────────────────────────────────┤
   │  실행 로그 (Executions)                       │
   └─────────────────────────────────────────────┘
   ```

4. 워크플로우 이름을 변경합니다
   - 상단의 "My workflow" 텍스트를 클릭
   - **"실습1_HelloWorld"** 로 변경
   - Enter 키를 누릅니다

> 💡 **TIP**: 워크플로우 이름에 번호를 붙이면 나중에 찾기 쉽습니다!

---

### Step 3: Manual Trigger 노드 추가

1. 캔버스 중앙의 **[ + ]** 버튼을 클릭합니다
   - 또는 캔버스 빈 곳을 더블클릭해도 됩니다
   - 또는 키보드 `Tab` 키를 눌러도 됩니다

2. 검색창에 **"Manual"** 을 입력합니다

3. **"Manual Trigger"** 를 클릭하여 추가합니다

4. 캔버스에 Manual Trigger 노드가 나타납니다

   ```
   ┌──────────────┐
   │ ▶            │
   │ Manual       │──── ●  (출력 포인트)
   │ Trigger      │
   └──────────────┘
   ```

> ✅ **확인**: 캔버스에 "Manual Trigger" 노드가 보이면 성공!

---

### Step 4: Edit Fields (Set) 노드 추가

1. Manual Trigger 노드 우측의 **작은 원(●)** 을 클릭합니다
   - 또는 Manual Trigger 노드 우측의 **[ + ]** 버튼을 클릭합니다

2. 검색창에 **"Edit Fields"** 를 입력합니다

3. **"Edit Fields (Set)"** 를 클릭하여 추가합니다

4. 두 노드가 자동으로 연결됩니다

   ```
   ┌──────────────┐        ┌──────────────┐
   │ ▶ Manual     │───────▶│ Edit Fields  │
   │   Trigger    │        │ (Set)        │
   └──────────────┘        └──────────────┘
   ```

---

### Step 5: Edit Fields 노드 설정

1. **Edit Fields (Set)** 노드를 더블클릭하여 설정 창을 엽니다

2. **"Add Field"** 버튼을 클릭합니다

3. 첫 번째 필드를 설정합니다:
   - **Name** (필드 이름): `message`
   - **Type**: String (기본값)
   - **Value** (값): `Hello n8n! 첫 번째 워크플로우입니다.`

4. **"Add Field"** 버튼을 다시 클릭합니다

5. 두 번째 필드를 설정합니다:
   - **Name**: `name`
   - **Type**: String
   - **Value**: `(본인 이름 입력)` 예: `김정인`

6. **"Add Field"** 버튼을 다시 클릭합니다

7. 세 번째 필드를 설정합니다:
   - **Name**: `timestamp`
   - **Type**: String
   - **Value**: `{{ $now.format('YYYY-MM-DD HH:mm:ss') }}`

   > 💡 `{{ }}` 안에 표현식(Expression)을 넣으면 동적 값을 생성할 수 있습니다!

8. 설정 창 우측 상단의 **"Back to canvas"** (← 아이콘)를 클릭하여 캔버스로 돌아갑니다

---

### Step 6: 워크플로우 실행

1. 캔버스 상단의 **"Test workflow"** (▶ 버튼)를 클릭합니다

2. 실행이 완료되면 각 노드 위에 **초록색 체크(✓)** 마크가 나타납니다

   ```
   ┌──────────────┐        ┌──────────────┐
   │ ✓ Manual     │───────▶│ ✓ Edit Fields│
   │   Trigger    │        │   (Set)      │
   └──────────────┘        └──────────────┘
   ```

3. **Edit Fields 노드**를 클릭하여 결과를 확인합니다

4. **"Output"** 탭을 클릭합니다

5. 아래와 같은 JSON 결과가 보여야 합니다:

   ```json
   [
     {
       "message": "Hello n8n! 첫 번째 워크플로우입니다.",
       "name": "김정인",
       "timestamp": "2026-03-22 14:30:00"
     }
   ]
   ```

> ✅ **확인**: Output에 3개 필드(message, name, timestamp)가 정상적으로 보이면 성공!

---

### Step 7: 워크플로우 저장

1. 키보드 **Ctrl + S** (Mac: Cmd + S) 를 눌러 저장합니다
   - 또는 상단의 **"Save"** 버튼을 클릭합니다

2. 좌측 메뉴 **"Workflows"** 를 클릭하여 목록에서 방금 만든 워크플로우를 확인합니다

> 💡 **습관**: 작업 중 수시로 Ctrl+S를 눌러 저장하세요! 브라우저가 닫히면 저장 안 된 내용은 사라집니다.

## 노드 기본 조작 연습

### 노드 추가
- 캔버스 빈 곳 **더블클릭** 또는 **Tab** 키

### 노드 연결
- 노드 우측의 **작은 원(●)** 을 드래그하여 다음 노드로 끌어다 놓기

### 노드 복사
- 노드 선택 후 **Ctrl + D** (Mac: Cmd + D)

### 노드 삭제
- 노드 선택 후 **Delete** 키 또는 **Backspace** 키

- 노드 **우클릭** → **"Deactivate"** 선택

- 연결선을 클릭하여 선택 후 **Delete** 키

### 되돌리기
- **Ctrl + Z** (Mac: Cmd + Z)

## 핵심 개념 정리

### 1. 노드 (Node)
작업의 최소 단위입니다. 3가지 유형이 있습니다:
- **트리거 노드**: 워크플로우의 시작점 (Manual, Schedule, Webhook 등)
- **액션 노드**: 실제 작업 수행 (Google Sheets, Gmail, Slack 등)
- **로직 노드**: 흐름 제어 (IF, Switch, Merge 등)

### 2. 엣지 (Edge / Connection)
노드와 노드를 연결하는 화살표 선입니다.
- 데이터가 이 선을 따라 왼쪽에서 오른쪽으로 흐릅니다
- 하나의 노드에서 여러 노드로 연결 가능합니다 (분기)

### 3. 실행 (Execution)
워크플로우가 한 번 돌아가는 단위입니다.
- 좌측 메뉴 "Executions" 에서 실행 이력을 확인할 수 있습니다
- 각 실행의 성공/실패 상태와 데이터를 확인할 수 있습니다

### 4. JSON 데이터
n8n은 모든 데이터를 JSON 형식으로 전달합니다.
```json
{
  "key": "value",
  "숫자": 123,
  "배열": ["항목1", "항목2"],
  "중첩": {
    "내부키": "내부값"
  }
}
```

### 5. 표현식 (Expression)
`{{ }}` 안에 동적 값을 넣을 수 있습니다.
- `{{ $json.필드명 }}` : 이전 노드의 출력 데이터 참조
- `{{ $now }}` : 현재 시간
- `{{ $input.all() }}` : 모든 입력 데이터

## 미니 퀴즈

1. n8n에서 워크플로우의 시작점 역할을 하는 노드 유형은?
   → **트리거 노드 (Trigger Node)**

2. 노드와 노드를 연결하는 선을 무엇이라 하나요?
   → **엣지 (Edge) 또는 Connection**

3. n8n에서 데이터를 전달하는 표준 형식은?
   → **JSON**

4. 워크플로우가 한 번 실행되는 단위를 무엇이라 하나요?
   → **Execution (실행)**

5. `{{ $json.name }}` 표현식의 의미는?
   → **이전 노드 출력 데이터에서 name 필드 값을 가져온다**
