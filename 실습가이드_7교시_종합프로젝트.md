---

## 프로젝트 개요

### 미션
> **"우리 팀의 교육 업무를 n8n + AI로 자동화하라!"**

| 항목 | 내용 |
|------|------|

## 프로젝트 시나리오 (팀별 1개 선택)

---

### 시나리오 A: 신규 과정 런칭 자동화

**상황**: 새로운 교육 과정을 오픈할 때마다 홍보 이메일 발송, 신청 접수, 수강생 관리를 수작업으로 처리하고 있어 시간이 많이 소요됩니다.

**자동화 목표**: 과정 정보를 입력하면 홍보부터 신청 접수까지 자동 처리

**추천 워크플로우 구조**:
```
Manual Trigger (과정 정보 입력)
    → Google Sheets (과정 DB에 등록)
    → OpenAI (홍보 이메일 문구 AI 자동 생성)
    → Google Sheets (기존 수강생 목록 조회)
    → Gmail (홍보 이메일 일괄 발송)
    → Slack (관리자 알림: "신규 과정 오픈 완료")
```

**필요한 시트**:
- "과정_DB": 과정명, 강사, 시작일, 수강료, 정원
- "수강생_명단": 기존 수강생 (이름, 이메일)

**핵심 기술**: OpenAI (마케팅 문구 생성), Gmail (일괄 발송), Sheets

---

### 시나리오 B: 수강생 학습 여정 자동화

**상황**: 수강 신청 → 환영 메일 → 수업 리마인더 → 수업 후 설문 → 수료증 발급까지의 전체 과정을 자동화하고 싶습니다.

**자동화 목표**: 수강 신청 시점부터 수료까지 전체 여정을 자동 관리

**추천 워크플로우 구조**:
```
Webhook (수강 신청 수신)
    → Edit Fields (데이터 정리)
    → Google Sheets (신청 DB 저장)
    → Gmail (환영 이메일 발송)
    → OpenAI (개인 맞춤형 학습 안내 문구 생성)
    → Gmail (맞춤형 안내 이메일 발송)
    → Slack (관리자 알림)
```

**추가 워크플로우 (Schedule 기반)**:
```
Schedule Trigger (매일 09:00)
    → Sheets (내일 수업 조회)
    → IF (수업 있는 경우)
    → Gmail (리마인더 발송)
```

**핵심 기술**: Webhook, Schedule, OpenAI (맞춤형 안내), Gmail

---

### 시나리오 C: AI 교육 어시스턴트

**상황**: 수강생 질문 대응, 수업 내용 요약, 학습 자료 추천을 강사 혼자서 처리하기 어렵습니다.

**자동화 목표**: AI가 FAQ 응답, 수업 요약, 자료 추천을 자동 처리

**추천 워크플로우 구조**:
```
Webhook (질문 수신)
    → Google Sheets (FAQ 데이터 로드)
    → Code (FAQ + 질문 결합)
    → OpenAI (답변 생성)
    → IF (답변 가능 여부)
        ├── 가능 → Respond to Webhook (AI 답변)
        └── 불가 → Gmail (강사에게 전달) → Respond to Webhook (안내 메시지)
    → Google Sheets (Q&A 로그 저장)
```

**보너스 기능**: 주간 Q&A 통계 자동 집계
```
Schedule Trigger (매주 금요일)
    → Sheets (이번 주 Q&A 로그 조회)
    → OpenAI (주간 질문 트렌드 분석)
    → Gmail (강사에게 주간 리포트 발송)
```

**핵심 기술**: Webhook, OpenAI (FAQ 기반 답변), Code, IF

---

### 시나리오 D: 교육 운영 대시보드 자동화

**상황**: 수강 현황, 만족도, 출결 데이터가 여러 시트에 흩어져 있어 주간 리포트 작성에 반나절이 소요됩니다.

**자동화 목표**: 데이터를 자동 집계하고 AI가 분석한 리포트를 자동 생성·발송

**추천 워크플로우 구조**:
```
Schedule Trigger (매주 월요일 09:00) 또는 Manual Trigger
    → Google Sheets (수강 현황 데이터 읽기)
    → Google Sheets (피드백 데이터 읽기)
    → Code (통합 통계 계산)
    → OpenAI (주간 분석 리포트 AI 생성)
    → Google Sheets (리포트 시트에 저장)
    → Gmail (관리자에게 리포트 이메일 발송)
    → Slack (주간 리포트 요약 공유)
```

**핵심 기술**: 다중 Sheets 조회, Code (통계 계산), OpenAI (분석), Gmail

## 프로젝트 진행 가이드

### Phase 1: 설계 (10분)

1. **시나리오 선택** — 팀원과 논의하여 A/B/C/D 중 1개 선택
   - Day 1 과제로 정리한 업무를 반영해도 좋습니다

2. **As-Is / To-Be 정리**:
   ```
   As-Is (현재): 수동으로 어떻게 처리하고 있는지
   To-Be (목표): n8n + AI로 어떻게 자동화할 것인지
   ```

3. **워크플로우 흐름도 스케치** (종이 또는 화이트보드):
   ```
   [트리거] → [노드1] → [노드2] → ... → [출력]
   ```

4. **필요 자원 목록**:
   - Google Sheets: 어떤 시트 필요?
   - Credential: 어떤 인증 필요?
   - AI 프롬프트: System Prompt 초안

5. **역할 분담**:
   - 설계자: 전체 구조 설계, 데이터 흐름 관리
   - 구현자: n8n에서 워크플로우 구축
   - 테스터: 테스트 데이터 준비, 실행 검증
   - 발표자: 발표 자료 준비, 데모 시나리오 작성

---

### Phase 2: 구현 (35분)

#### 구현 순서 (권장)

1. **시트 준비 (5분)**
   - 필요한 Google Sheets 생성 & 샘플 데이터 입력

2. **트리거 설정 (3분)**
   - Webhook 또는 Manual Trigger부터 시작

3. **데이터 소스 연결 (7분)**
   - Google Sheets 읽기 노드 추가 & 테스트

4. **데이터 가공 (5분)**
   - Edit Fields / Code 노드로 데이터 변환

5. **AI 연동 (10분)**
   - OpenAI 노드 추가 & 프롬프트 작성·테스트

6. **출력 설정 (5분)**
   - Gmail / Slack / Respond to Webhook 연결

> ⚠️ **중요**: 노드를 하나 추가할 때마다 **"Test step"** 으로 해당 노드만 테스트하세요!
> 전체를 다 연결한 후에 테스트하면 디버깅이 어려워집니다.

#### 구현 중 체크 포인트

- [ ] 트리거 노드가 정상 동작하는가?
- [ ] Google Sheets 읽기가 정상인가? (데이터 확인)
- [ ] Edit Fields에서 필드명이 올바른가?
- [ ] AI 프롬프트가 원하는 결과를 생성하는가?
- [ ] 이메일/Slack 발송이 정상인가?
- [ ] 전체 워크플로우 end-to-end 테스트 통과?

---

### Phase 3: 테스트 (10분)

1. **정상 케이스 테스트** — 올바른 데이터로 3회 실행
2. **엣지 케이스 테스트** — 빈 데이터, 잘못된 형식 등
3. **에러 상황 테스트** — 의도적 오류 발생 시 워크플로우 동작 확인
4. **테스트 결과 정리** — 스크린샷 캡처

---

### Phase 4: 발표 준비 (5분)

1. 워크플로우 전체 모습 스크린샷
2. 주요 노드 설정 스크린샷
3. 실행 결과 스크린샷 (Output 데이터, 이메일 수신 등)
4. 발표 순서 정리: 문제 → 해결 → 데모 → 기대효과
